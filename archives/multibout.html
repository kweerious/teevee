Bout Summary

Dev Note: The Rinxter grid has an attribute called data-bout-id where you'd drop in the bout and tournament id for the page.

<div id="rinxter" data-bout-id="173" data-tournament-id="3" class="span5"></div>
Then there's a list for the tabs and in the corresponding tab you drop in the grid you need. Example:
<div id="bout_summary"></div>
for the Bout Summary grid in the main tab.

For multi-event pages (tournaments), add the bouts for the page in the Bouts tab marked up with their bout id and the class new_bout on the anchor:

<div class="new_bout" data-bout-id="452"></div>
When these links are selected, the bout data will update with the selected bout information and update the grids and full stats link. Note: they should include the link to the full stats page as a fallback for those with no javascript.

Finally, there's the Auto Refresh toggle at the bottom of the tabs that shows up during live events (not completed/archived ones). I've defaulted it to off and left it to the user to switch on. I think this will save on bandwidth and empowers the user to use it or turn it off if they don't like it.

Note: Top Penalties doesn't have any data for bouts with no penalty tracking. This is an issue for live event stats slated for the Big 5 which will only have lineups and score live (at least that's what was announced).
